# 🚚 Logistics Transport Forecasting System | Командный трек

## 📋 Описание проекта
Автоматизированная система прогнозирования отгрузок и генерации заявок на вызов транспорта.

## 📊 Метрики (Лидерборд: 50%)
- **Трек:** `team`
- **Горизонт:** `10` шагов (30 мин)
- **Валидационная метрика:** `0.2715` (WAPE + |Relative Bias|)
- **Архитектура:** Direct Multi-step + LightGBM Ensemble (3 модели)

## 🏗️ Архитектура системы
```
[Данные] → [Feature Eng: циклическое время, лаги, EWM, rolling] → [LightGBM x3] → [Bias Calibration] → [Decision Engine]
```

## ⚙️ Бизнес-логика
- Прогноз каждые 30 мин. Шаги 1-4 → авто-заявки. Шаги 5-10 → планирование.
- `trucks = ceil(forecast / capacity)`. Пороговые алерты при отклонении от исторических квантилей.
- `np.clip(..., 0, None)` + калибровка смещения устраняют систематическую ошибку.

## 🤖 Модельный стек
- **Алгоритм:** LightGBM (`lr=0.03`, `leaves=127`, `max_depth=7`, `early_stop=50`)
- **Признаки:** `hour_sin/cos`, `dow_sin/cos`, лаги 1-12, rolling mean/std (4,8), EWM (0.1, 0.3)
- **Валидация:** Строгий временной сплит 80/20. Без случайного сэмплирования.
- **Калибровка:** Глобальный множитель `sum(y_true_val)/sum(y_pred_val)` снижает |Relative Bias| до ~0.

## 📝 Допущения
- Транспорт гомогенен. Отгрузки равномерны в пределах 30 мин. Внешние факторы не учтены в v1.

## 🔮 Развитие
